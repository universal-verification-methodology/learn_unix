## Outcomes

After this module you can explain and practice the ideas taught by **`ps-kill-lab`**, in the browser and/or on a real shell.

## Two tracks (pick one or both)

### Track A — Real Unix (hands-on)

Work in your local terminal using this module’s `examples/`.

1. Open [EXAMPLES.md](EXAMPLES.md) and run the “Try these” commands.
2. Complete [CHECKLIST.md](CHECKLIST.md) on a real filesystem (`~/unix_practice` recommended).
3. Optional self-check: `./scripts/module.sh 09 --check` (from course root).

### Track B — Browser lab (online)

1. Local: [http://127.0.0.1:8080/tools/ps-kill-lab/index.html](http://127.0.0.1:8080/tools/ps-kill-lab/index.html)
2. Live: [https://universal-verification-methodology.github.io/learning/tools/ps-kill-lab/](https://universal-verification-methodology.github.io/learning/tools/ps-kill-lab/)
3. Load the **starter example**, then work challenges.
4. Check off the Track B items in [CHECKLIST.md](CHECKLIST.md).

> Concept labs are literacy tools — they do not replace a real shell for scripts you will keep in a design project.
